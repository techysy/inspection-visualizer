# 📋 更新日志

## [3.3.0] - 2026-07-16

### ✨ 新增

- **截图预览页面**：`/backup-gallery` 按日期分组展示备份截图缩略图，支持 Lightbox 全屏预览（左右箭头/键盘切换）
  - API：`/api/backup/gallery` 返回备份文件夹和图片列表
  - 管理员导航栏新增「截图预览」入口
- **增量计算（按天日均）**：仪表盘类型新增独立的「启用增量计算」开关，可与任意计算类型（百分比/求和/差值/自定义公式）同时使用
  - 配置项：增量指标名称、单位、对比源字段、图表最大值
  - 保存记录时自动与上一条记录对比，首条记录增量为 0
  - 间隔超过 1 天时自动除以天数，存入日均值（如周末 3 天间隔自动÷3）
  - 自动创建指标配置并标记为「计算类」，不可手动编辑
- **数据维护增量回填**：数据维护弹窗新增「补全增量指标」选项，支持批量回填历史增量数据（同样按天数平滑）
- **标签自动清理**：保存仪表盘类型时自动从标签映射中移除计算类指标的 result_name，避免脏数据

### 🐛 修复

- **增量指标不同步到对象**：`_sync_virtual_metrics_for_dtype` 中 `single_config` 替换 `calc_configs` 而非合并，导致增量指标丢失
- **增量指标不标记为计算类**：`_get_virtual_metric_keys_for_type` 同样存在替换问题，增量 result_name 未加入虚拟指标集合
- **保存后增量配置丢失**：PUT/POST handler 未保存 `calc_configs` 字段
- **改名后历史数据残留**：编辑类型修改 calc result_name 后，自动同步更新历史记录中的 key 和 ObjectMetric，清理旧指标
- **改名后标签脏数据**：保存时自动清理 labels 中的 calc result_name
- **指标 max_value 自动更新导致长小数**：移除保存记录时自动将 max_value 设为 `值×1.5` 的逻辑，新指标默认 max=100
- **图表 tooltip 缺少单位**：`mc.unit === 'w'` 改为通用单位显示
- **图表 0 值不显示**：`parseFloat("0") || null` 将 0 转为 null，改为 `isNaN` 判断
- **calc_config 替换 calc_configs**：`_get_virtual_metric_keys`、`_get_virtual_metric_keys_for_type`、`_sync_virtual_metrics_for_dtype` 三处统一修复为合并

### 🔧 优化

- **增量计算独立于计算类型**：从计算类型下拉框移出，作为独立开关，可与任意计算类型组合
- **源字段下拉过滤**：增量的「对比源字段」下拉框自动排除计算类指标的 result_name
- **UI 描述更新**：增量计算说明改为「首条按0计算」，数据维护回填说明改为「逐条与上一条对比」

### 🔧 优化

- **对象详情页布局优化**：巡检统计和巡检历史记录标题恢复原始布局，计数标签靠右显示

## [3.1.0] - 2026-07-07

### 🐛 修复

- **闪黑问题**：首屏 `data-theme="dark"` 硬编码，主题 JS 在底部执行导致浅色/自动主题用户先渲染黑底再切换，改为 `<head>` 同步阻塞脚本提前设置主题
- **扩展登录按钮过宽**：`.btn` 基类 `width: 100%` 导致登录按钮占满整行，输入框被挤到几像素，加 `width: auto` 覆盖
- **扩展登录回车失效**：密码框未绑定回车事件，`doLogin()` 重构后旧 click handler 体残留导致语法错误
- **扩展登录后提示残留**：登录成功未清除"需要登录才能使用"状态提示
- **`/object/<id>` 500 错误**：`session` 变量名在函数内被 `SessionLocal()` 覆盖导致 `UnboundLocalError`，重命名为 `db`
- **扩展保存后跳转首页**：`lastSavedObjectId` 从 OCR 结果取 `object_id`（不存在），改为用 `/api/save` 返回的 `data.object_id`
- **导航栏 active 样式**：`border-radius: 8px 8px 0 0` + `border-bottom` 组合视觉不自然，改为 `::after` 下划线

### ✨ 新增

- **数据鉴权过滤**：普通人员只能看到自己的巡检记录（首页、对象详情页、API 历史接口均按 `inspector_id` 过滤）
- **导航栏权限控制**：非管理员隐藏「导入导出」「对象管理」入口及后端路由保护
- **首页排序优化**：已巡检对象在前、同类设备分组、未巡检在后
- **人员管理添加弹窗**：行内表单改为 Modal 弹窗添加人员
- **Chrome 扩展: 密码框回车登录**：绑定 keydown 事件，回车触发登录

### 🔧 优化

- **导航栏重构**：主题切换按钮和用户信息移出 `<ul>`，与导航链接分离，`me-auto` 实现导航靠左工具区靠右
- **首页未巡检显示**：去掉 `{% if item.latest_record %}` 过滤，无记录时显示「未巡检」标签
- **扩展登录横幅**：加 `overflow: hidden` 和 `min-width: 0` 防止元素溢出边界

- **Chrome 扩展同步更新**：弹窗登录栏增加用户名输入框，`background.js` 携带用户名调用 API

### ✨ 新增

- **虚拟指标管理**：仪表盘类型的 `calc_config` 自动创建/同步虚拟 `ObjectMetric`，不可删除仅可编辑阈值
- **虚拟指标按类型过滤**：`calc_config.result_name` 仅标记当前类型的虚拟指标，避免跨类型误标
- **OCR 诊断日志**：图片尺寸、OCR 耗时、结果条数等调试信息
- **导航栏用户信息**：显示当前登录人员姓名，增加退出登录按钮

### 🐛 修复

- **Chrome 扩展 Service Worker**：`cropImage` 中 `FileReader` 不可用，改为 `arrayBuffer` + 分块 `btoa` 编码
- **弹窗 JS 语法错误**：`popup.js` 中 `showResultItems` 外层 `forEach` 缺失闭合 `});`，导致整个脚本不执行
- **上传识别空结果**：OCR 对上传截图返回空（60ms），添加日志诊断图片尺寸过小问题
- **前端自动匹配**：`point_name` 为空但有 `region_name` 时，自动用位置名作为 `point_name`，确保 `hasNameMatch` 为 true
- **百分比值编辑**：弹窗中百分比输入框用 `parseFloat.replace` 提取数字，编辑后保留 `%` 后缀
- **对象格式公式**：`{expr, decimal_places, suffix}` 格式后缀 `%` 正确存储为 `"66.73%"`
- **`ObjectMetric` 创建异常**：`float()` 前先 `rstrip('%')`，避免百分比值转换失败
- **日志轮转命名**：`log_handler.namer` 使轮转文件名扩展名在日期后

### 🔧 优化

- **Chrome 扩展快捷键**：新增 `Ctrl+Shift+S` 快速启动框选截图
- **仪表盘类型保存**：保存时自动同步虚拟指标，原子写入 `dashboard_types.json`
- **详情页 `%` 显示**：`fmtVal` 保留 `%` 后缀，图表 Y 轴/tooltip 添加 `%`
- **保存记录自动关联人员**：登录后的巡检人员自动填入巡检记录 `inspector_id`
