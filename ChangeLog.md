# 📋 更新日志

## [2.5.0] - 2026-07-03

### ✨ 新增

- **计算类别补全**：数据维护新增计算补全功能，支持百分比、差值、求和、自定义公式四种计算方式
- **对象列表筛选**：对象管理页面输入名称/位置/分类时实时筛选显示匹配的巡检对象
- **日志按日轮转**：应用日志存放在 `log/` 目录，按天自动切片，保留最近 30 天

### 🔧 优化

- **数据维护弹窗 UI**：统一使用 Bootstrap 模态框样式，与编辑弹窗保持一致
- **删除确认弹窗**：修复弹窗 z-index 层级问题，确保遮罩层正确覆盖页面内容

### 🐛 修复

- 修复数据维护弹窗 `display` 属性重复导致始终显示的问题
- 修复删除确认弹窗被页面元素遮挡的问题

## [2.4.0] - 2026-07-02

### ✨ 新增

- **历史记录按周分组**：巡检详情页历史记录按「本周/上周/更早」分组，上周及更早自动收起，可点击展开
- **导出离线值自动计算**：导出 Excel 时离线值由 `总数 - 在线` 自动计算
- **对象创建自动补全**：添加巡检对象时，名称、位置、分类输入框支持搜索自动补全，显示推荐建议

### 🔧 优化

- **对象排序持久化**：对象管理页面拖拽排序结果保存后，刷新页面按已保存的顺序显示（`sort_order` 优先于 `location, name`）
- **导出总数取值**：总数优先使用 OCR 识别的 `total` 值，其次使用「在线」指标的 `max_value`
- **导出在线率取值**：优先使用已有的「在线率」指标值，无则按 `在线 / 总数` 计算
- **导出对象匹配**：同名不同位置的巡检对象（如城区/郊区垃圾站）导出时按 `name + location` 精确匹配
- **巡检结果趋势图**：从时间轴改为分类轴，消除周末空白间隙，确保最新日期点始终可见
- **历史记录状态对齐**：状态 badge 固定宽度，避免二字/三字状态导致巡检人位置错位

### 🐛 修复

- 修复导出 Excel 时 `offline_val` 引用未定义 `total` 变量导致的文件无法打开
- 修复趋势图 `afterBuildTicks` 过滤周末后 x 轴仍预留空白宽度的问题
- 修复导出时仅按名称匹配对象，同名不同位置对象被错误归并的问题

## [2.3.0] - 2026-06-29

- **全局变量配置**：OCR 管理页面新增全局变量模块，可配置「跳过关键词」（提取位置时忽略的文本）
- **不使用位置匹配**：仪表盘类型新增「不使用位置匹配」选项，启用后仅通过类型分类匹配巡检对象，忽略位置

- **日志文件**：运行日志写入 `app.log`，便于后台排查问题

### 🔧 优化

- **OCR 括号格式支持**：标签数值解析支持 `在线(30)` 和 `全部（50）` 等括号格式
- **位置匹配优先级**：OCR 文本位置优先于文件名位置，提高匹配准确率
- **状态判定关联阈值**：仪表盘类型的 `result_rules` 现在正确关联到识别结果状态标签
- **UI 布局优化**：识别结果卡片中位置和分类合并显示（如「三期垃圾站 监控」）
- **详情页统计**：新增「需关注」计数，两栏卡片等高显示
- **仪表盘类型标签**：labels 使用英文 key 匹配 result_rules，修正阈值判断不生效问题

### 🐛 修复

- 修复计数模式相关功能移除后的缩进错误导致服务无法启动
- 修复仪表盘类型编辑时「不使用位置匹配」选项未保存的问题
- 修复全局变量配置区域不显示输入框的问题
- 修复识别结果状态标签未关联 result_rules 阈值的问题

### 🗑️ 移除

- 移除计数模式（count_mode）相关功能：OCR 解析、仪表盘类型配置、对象管理页面日列表入口、批量导入计数模式 API
- 移除 `LOCATION_ALIAS` 硬编码位置映射和 `normalize_location` 函数

## [2.2.0] - 2026-06-28

### ✨ 新增

- **仪表盘类型分类管理**：仪表盘类型新增 `category` 字段，支持分类筛选和关联
- **巡检对象分类联动**：巡检对象的「分类」字段与仪表盘类型分类联动，使用 `<select>` 下拉选择，支持新建分类
- **快速创建巡检对象**：截图识别后若未匹配到已有对象，显示下拉列表 + 新建按钮，弹窗创建对象并自动关联
- **自动同步指标**：创建巡检对象时自动从关联的仪表盘类型同步指标配置
- **指标 max_value 自动更新**：保存巡检记录时，若指标值超过当前 max_value 的一半，自动更新为当前值的 1.5 倍
- **Toast 通知**：操作提示从 `alert()` 改为顶部居中气泡通知，3.5 秒自动消失
- **新建分类功能**：仪表盘类型和巡检对象的分类下拉框均支持「+ 新建分类」选项
- **新增 API**：
  - `GET /api/dashboard-types/categories` — 获取所有分类
  - `POST /api/objects/<id>/sync-metrics` — 从仪表盘类型同步指标
  - `POST /api/dashboard-types` — 添加仪表盘类型（含分类）
  - `PUT /api/dashboard-types/<id>` — 更新仪表盘类型（含分类）

### 🔧 优化

- **OCR 匹配逻辑优化**：去掉仅凭 `device_type` 相同就匹配的逻辑，改为必须有名称相似度才匹配，避免误匹配
- **仪表盘类型卡片**：类型卡片显示分类标签，跟随主题色
- **图表 Y 轴**：max_value 计算从 2 倍改为 1.5 倍，避免图表空间浪费
- **单位自动判断**：仅当指标值 ≥ 10000 时自动添加 `w`（万）单位，小数值不再强制加单位
- **巡检对象管理**：类型字段改为分类下拉框，与仪表盘类型分类联动
- **OCR 管理页面**：分类字段改为下拉选择，支持新建分类

### 🐛 修复

- 修复仪表盘类型编辑/新建时分类字段未保存的问题
- 修复 OCR 识别后自动匹配到不相关对象的问题
- 修复快速创建巡检对象时分类错误使用仪表盘类型名称而非分类的问题
- 修复操作提示使用 `alert()` 弹窗的问题，改为 Toast 通知
- 修复同步指标后图表 Y 轴溢出的问题

## 🚀 [2.0.0] - 2026-06-23

### 🔄 重构：胶卷价格追踪 → IT运维巡检数据可视化

基于 film-price-tracker 项目重构为 IT运维巡检记录管理工具。

### ⚠️ 重大变更

#### 🗄️ 数据模型重构
- `Film` → `InspectionObject`（巡检对象）
- `PriceHistory` → `InspectionRecord`（巡检记录）
- `TaobaoStore` → `Inspector`（巡检人员）
- 数据库表：`inspection_points` → `inspection_objects`
- 数据库文件：`film_prices.db` → `inspection_data.db`

#### 🛤️ 路由变更
- `/points` → `/objects`
- `/point/<id>` → `/object/<id>`
- `/points/add` → `/objects/add`
- `/points/edit/<id>` → `/objects/edit/<id>`
- `/points/delete/<id>` → `/objects/delete/<id>`

#### 📝 命名变更
| 原命名 | 新命名 |
|--------|--------|
| 点位 | 巡检对象 |
| 点位管理 | 对象管理 |
| 设备类型 | 类型 |

### ✨ 新增功能

- 📸 **截图识别**：OCR 自动提取巡检表单数据，支持 Ctrl+V 粘贴、拖拽上传
- 📦 **批量导入** `/import`：支持 Markdown 表格和 CSV 格式批量导入对象、人员、巡检记录
- 📍 **对象管理**：管理巡检对象，支持位置、类型分类
- 👥 **人员管理**：管理巡检人员/班组信息
- 📊 **趋势图表**：Chart.js 折线图展示巡检结果历史趋势
- 🎨 **主题切换**：深色/浅色/跟随系统三档切换
- 🔍 **双维度筛选**：按位置 + 类型组合筛选巡检对象
- 💾 **数据导出**：JSON 和自包含 HTML 两种格式

### 🛠️ 技术栈

- 🐍 Python 3.8+
- 🌐 Flask + SQLAlchemy + SQLite
- 🔤 RapidOCR (ONNX Runtime) - 图片文字识别
- 🎨 Bootstrap 5 + Chart.js - 前端
