# 🔌 巡检截图识别 Chrome 扩展

浏览器插件，直接在任意页面框选区域截图并 OCR 识别巡检数据，无需手动打开系统上传截图。

## 📦 安装

1. 打开 Chrome，地址栏输入 `chrome://extensions`
2. 右上角打开「开发者模式」
3. 点击「加载已解压的扩展程序」
4. 选择本项目中的 `chrome-extension` 文件夹
5. 扩展图标出现在工具栏，建议点击 📌 固定

## 🚀 使用方法

### 方式一：框选截图（推荐）

1. 打开巡检表单或仪表盘页面
2. 点击工具栏扩展图标
3. 点击「框选截图识别」
4. 在页面上拖拽框选需要识别的区域
5. 松开后自动截图 → OCR 识别 → 显示结果
6. 可直接编辑识别到的数值
7. 点击「保存到系统」写入数据库（截图自动备份到服务器）

### 快捷键

- `Ctrl+Shift+S` — 快速启动框选截图
- `Esc` — 取消当前框选

## ⚙️ 前提条件

- 主系统需运行在 `http://localhost:5001`
- 如需通过扩展保存数据，确保系统已启动且可访问
- 系统需先通过浏览器登录（登录密码在根目录 `.env` 文件中配置）

## 📁 文件说明

| 文件 | 说明 |
|------|------|
| `manifest.json` | 扩展配置（Manifest V3） |
| `popup.html/js` | 点击扩展图标弹出的 UI |
| `content.js/css` | 注入页面的框选覆盖层 |
| `background.js` | 后台消息中转 |
| `icons/` | 扩展图标（16/48/128px） |

## 🔧 工作原理

```
点击扩展 → 注入 content.js → 用户框选区域
    ↓
captureVisibleTab 截取全屏 → canvas 裁剪选中区域
    ↓
POST /api/ocr → OCR 识别 → 返回结构化数据
    ↓
用户编辑确认 → POST /api/save → 保存到数据库 + 备份截图
```

## 🐛 常见问题

**Q: 框选后提示"无法注入脚本"**
A: 检查扩展权限是否完整，尝试刷新页面后重新框选。

**Q: 识别结果为空**
A: 确认截图区域包含巡检数据文字，系统需运行在 `localhost:5001`。

**Q: 保存失败**
A: 确认后端服务已启动，检查 `http://localhost:5001` 是否可访问。
